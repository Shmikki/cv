Об одном из проектов

  • Кроссбраузерный (IE8+, FF, Chrome) web-интерфейс для организаций партнеров.
    Эта система представляет собой полноценное web-приложение для создания и сопровождения заявок по кредитам.
    200 и более полей ввода

    В форме для подачи заявки может быть от десятка до двух сотен и более связанных между собой полей ввода.

    Для динамического создания анкеты было решено, что каждое поле должно уметь работать только с самим собой и не должно напрямую воздействовать на окружающие поля. Каждое поле умеет посылать во внешний мир сигнал о событии и прикрепленное к сигналу сообщение. Так же, оно умеет слушать сигналы от интересующих его полей и реагировать на них, перестраивая себя необходимым образом.

    Например, можно сказать, что поле A "подписывается на новости" от полей B, C и E. В случае, если поля B не существует, то ничего плохого не происходит, просто от него не приходят сигналы.

    Работа с КЛАДР

    В различных анкетах может быть до десятка групп адресов, включающих в себя поля "регион", "район", "город", ..., "корпус", "строение", "квартира".

    Для того чтобы уменьшить ошибки пользователя при вводе адресов используется база адресов КЛАДР. При вводе, для автодополнения предлагается список подходящих объектов. Кроме этого, если например ввести сразу город, и выбрать его из предлагаемого списка, то поля, находящиеся выше по иерархии (регион и район) заполнятся автоматически.

    Проверка и форматирование введенных данных

    Для данных существуют единые правила проверки на клиентской и серверной стороне.

    Каждое поле содержит две функции проверки. Первая функция отвечает за форматирование данных (удаление лишних сущностей и/или добавление нужных). Вторая за сравнение форматированных данных с регулярным выражением.

    Например, на клиентской стороне, при вводе номера телефона 9876543210 он налету приводится к виду (987)654-32-10. Т.е. пользователь сразу видит форматированный ввод. При этом одной из сложностей было управление курсором при добавлении символов в вводимую строку.

    Универсальный вывод актуальных подсказок

    Подсказка для одного и того же поля может отличаться в зависимости от типа анкеты.

    Первоначально, в качестве инструкции к анкетам были прикреплены pdf документы. Это не удобно, т.к. для нового или не обученного сотрудника поиск инструкции по соответствующему полю занимает довольно много времени.

    Со временем инструкции для полей ввода и групп полей были перенесены в отдельное хранилище и интегрированы в саму анкеты. Существует два типа инструкций - короткая и полная. Короткая инструкция появляется при каждом попадании фокуса на поле ввода и содержит информацию о формате вводимых данных и краткое описание поля. Полная инструкция открывается по инициативе пользователя и содержит полную информацию о поле и его связи с другими полями.
